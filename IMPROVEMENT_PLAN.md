## 2. 주요 문제점

### 2.1 아키텍처 문제 (심각도: 높음)

#### 모놀리식 코드 구조
- `App.jsx`가 520줄로 모든 UI를 포함
- `src/components/`에 분리된 컴포넌트 파일이 존재하나 **미사용**
- 유지보수 및 확장이 어려운 구조

```
현재 구조:
src/
├── App.jsx          # 520줄, 모든 UI 포함 (문제)
├── components/
│   ├── Hero.jsx     # 미사용
│   ├── Navbar.jsx   # 미사용
│   ├── Footer.jsx   # 미사용
│   └── ...          # 미사용
```

#### 데이터 하드코딩
- 기본 데이터가 `App.jsx` 내부에 직접 작성됨
- Sanity CMS 연동 로직과 UI 로직이 혼재

### 2.2 디자인 시스템 불일치 (심각도: 중간)

| 문제 | 설명 |
|------|------|
| 컬러 토큰 불일치 | `tailwind.config.js`에 `gold` 정의 → 실제 코드는 `amber` 사용 |
| Primary 색상 미사용 | 커스텀 `primary` 팔레트 정의 → 실제로는 `slate` 직접 사용 |
| 일관성 부재 | 컴포넌트별 스타일 패턴이 다름 |

### 2.3 누락된 핵심 기능 (심각도: 높음)

| 기능 | 상태 | 프로 사이트 필수 여부 |
|------|------|----------------------|
| 검색 기능 | UI만 존재, 동작 안함 | 필수 |
| 매물 상세 페이지 | 없음 | 필수 |
| 회사소개 페이지 | 없음 | 필수 |
| 라우팅 시스템 | 없음 (단일 페이지) | 필수 |
| 지도 연동 | 없음 | 필수 |
| 실제 매물 이미지 | placeholder만 사용 | 필수 |
| 로딩 상태 UI | 없음 | 필수 |
| 에러 처리 UI | 없음 | 필수 |
| SEO 메타태그 | 없음 | 필수 |
| 접근성 (a11y) | 미흡 | 필수 |

### 2.4 UX 개선 필요 사항

| 영역 | 현재 상태 | 개선 방향 |
|------|-----------|-----------|
| 히어로 섹션 | 깔끔하지만 평범함 | 영상 배경, 더 임팩트 있는 CTA |
| 매물 카드 | 그라데이션 배경만 사용 | 실제 고품질 이미지 |
| 상담 섹션 | 기본 폼 | 채팅봇, 카카오톡 연동 |
| 모바일 UX | 기본 반응형 | 바텀 네비게이션, 터치 최적화 |
